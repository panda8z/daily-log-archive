# 二叉搜索树，平衡二叉树，红黑树的算法效率

| 操作 | 二叉查找树 | 平衡二叉树 | 红黑树  |
| ---- | ---------- | ---------- | ------- |
| 查找 | O(n)       | O(logn)    | Olog(n) |
| 插入 | O(n)       | O(logn)    | Olog(n) |
| 删除 | O(n)       | O(logn)    | Olog(n) |

# Olog(n)怎么算出来的

```
在一个树中查找一个数字，
第一次在根节点判断，第二次在第二层节点判断
以此类推，树的高度是多少就会判断多少次
树的高度和节点的关系就是以2为底，树的节点总数n的对数
```
